# 河洛战阵 / STZB Battle Prototype

这是一个静态 HTML/CSS/JS 原型，目标是做一个只关注战斗、抽卡、武将和战法配置的《率土之滨》风格 PVE 对战 demo。

## 当前功能

- 3 武将编队：大营、中军、前锋。
- 每名武将 10000 兵，最多 8 回合。
- 战斗一回合一回合推进，不会一次性出完整结果。
- 武将十连抽，带抽卡弹窗。
- 十连抽到 3 星和 4 星武将会自动转为狗粮并直接消失，不进入武将册，也不会出现在排兵布阵里；自动转狗粮时会同时解锁该武将对应的可拆战法。狗粮当前只记录数量，暂未接用途。
- 顶部 `重置` 按钮会清空本地存档，包括抽卡获得的武将、拆解解锁战法、编队和战斗记录，并回到初始阵容。
- 武将画像显示在武将册、战场单位和抽卡结果里。
- 点击武将可看详情：画像、阵营、兵种、星级、COST、攻击距离、四维、自带战法、可拆战法、传记。
- 点击战法可看详情：图标、品质、类型、兵种类型、有效距离、目标群体、发动率、效果和官方描述。
- 战法品质使用官网 `skill_extra.json` 里的 `zfQuality` 字段，UI 会显示官方 `S/A/B/C` 标注。
- 左侧新增 `战法集`，按品质/类型列出已抓取的全部战法；手写种子战法如果和官方战法同名或已建立别名映射，会自动回填官方品质、距离、目标、发动率、图标和描述。
- 点击弹窗外空白处可关闭武将详情和战法详情。
- 拆解已拥有武将解锁可拆战法，解锁后才会出现在编队战法下拉里。
- 可配战法遵循唯一占用规则：同一个战法在一支队伍里只能给一个武将装备一次。
- 兵种克制：骑克步、步克弓、弓克骑。
- 普通攻击按攻击距离判定目标；攻击距离不足时不会普攻，反击也必须能打到伤害来源。
- 战斗已加入准备回合、主动战法延迟准备、洞察免疫、怯战/犹豫/混乱/暴走控制、援护和分兵等基础状态规则。
- 同阵营和同兵种部队加成：至少 2 名同阵营或同兵种武将会触发对应属性加成。

- 战法资料库：`https://stzb.163.com/jineng_list.html`
- 战法详情数据：`https://g0.gph.netease.com/ngsocial/community/stzb/cfg/skill_extra.json?gameid=g10`
- 战法图标：`https://g0.gph.netease.com/ngsocial/community/stzb/front_end/img/jineng/tactics_00{1-4}.png?gameid=g10`
- 武将详情页：`https://stzb.163.com/herolist/{id}.html`

生成结果写入：

- `official-data.js`

当前一次生成结果：

- 官方武将：391 个
- 官方战法记录：837 条
- 游戏启动后会再合并少量手写原型武将/战法

当前 `hero_extra.json` 没有覆盖所有早期经典武将。项目里保留了少量手写种子武将用于开局体验；这些种子武将如果配置了可拆战法，必须来自对应官网详情页核对过的可拆字段，例如：

- `曹操`：`https://stzb.163.com/herolist/100023.html`，可拆 `远攻秘策`
- `刘备`：`https://stzb.163.com/herolist/100016.html`，可拆 `神兵天降`

官网页面提示数据仅供参考，以游戏内设定为准。

当前战法详情来自 `skill_extra.json`，会保存 `zfQuality`、`type`、`soldierType`、`distance`、`probability`、`targetShow`、`effect`、`desc` 和类型图标。旧的 `jineng_list.json` 只作为历史资料源记录，不再作为战法详情主来源。

## 数据脚本

重新抓取武将、战法和可拆战法：

```powershell
node .\scripts\scrape-stzb-official.mjs
```

脚本会：

- 抓取 `hero_extra.json`
- 抓取 `skill_extra.json`
- 访问每个 `herolist/{id}.html`
- 解析 `基础战法`
- 解析所有 `可拆战法` / `可拆战法2` 等块
- 保存官方 `zfQuality` 为战法 `grade`，并在 UI 显示为 `S级` / `A级` / `B级` / `C级`
- 生成 `official-data.js`

## 画像脚本

下载武将画像：

node .\scripts\download-stzb-portraits.mjs

## 拆解战法规则

拆解战法必须来自官网详情页里的 `可拆战法` / `可拆战法2` 等字段。

重要修正：

- 不再在缺少可拆数据时回退到武将自带战法。
- 不再把所有“某个武将的自带战法”从可配战法池里全局排除，因为这会误伤官网可拆/可研究战法。
- 拥有 1 张武将也可以拆解，不要求重复卡；拆解后该武将数量 -1。
- 如果被拆武将正在编队中，会从编队移除，并尽量用现有武将自动补位。
- 如果官网详情页给同一个武将列了多个可拆战法，拆解时会全部解锁到本地战法池。
- 抽卡时自动转为狗粮的 3 星和 4 星武将，也会按同一套可拆战法规则自动解锁战法。
- 手写种子武将也必须显式配置 `dismantles`，不能靠自带战法兜底。
- 少数官网条目本身会出现可拆战法和基础战法同名的情况；项目只按官网详情页记录，不自行猜测替换。

## 编队战法规则

- 每名武将有 1 个自带战法和最多 2 个可配战法。
- 自带战法不占用可配战法池。
- 可配战法在当前队伍中唯一占用：给某个武将装备后，其他武将的战法下拉里不会再出现该战法。
- 旧存档或异常状态里如果出现重复战法，加载/渲染时会保留先出现的一份，后续重复项会自动清空为 `未配置`。

## 战法等级与数值规则

当前原型不做战法升级系统，所有战法默认按满级处理。

- `skill_extra.json` 的 `probability` 如果是 `25%-35%` 这类范围，视为 1 级到 10 级成长，战斗判定取右侧满级值，例如 `25%-35%` 按 `35%` 发动。
- 战法详情弹窗保留原始范围，并额外标注“满级按 X%”。
- 抽样核对：`八门金锁` 官网数据为 `25%-36%`，攻略资料也写作 25% 发动、满级 36%；`侵掠如火` 官网 `desc` 已直接写满级 `20.0%` / `50.0%`，攻略资料写作 10.0% / 25.0% 到满级 20.0% / 50.0%。
- 因此当前把 `desc` 里的伤害率、增伤率、减伤率等数值按官网当前文本直接视为满级数值；如果后续发现某条官方文本仍是 1 级数值，再为该战法单独补结构化满级表。

## 攻击距离规则

- 战场按 6 个固定格计算距离：我方大营 0、中军 1、前锋 2；敌方前锋 3、中军 4、大营 5。
- 普通攻击只能选择距离小于等于当前攻击距离的存活敌军，并在可攻击目标里随机判定目标。
- `回马` 等反击也检查攻击距离；如果反击方够不到攻击来源，则不会造成反击伤害。
- 武将使用官网数据里的 `distance` 作为基础攻击距离；少量手写种子武将缺失距离时按兵种兜底：骑 2、步 2、弓 3。
- `远攻秘策` 已按描述实现为我军全体前 3 回合攻击 +20、谋略 +20、攻击距离 +1。
- 泛化官方战法优先使用 `skill_extra.json` 的 `distance` 筛选有效距离；缺失时才回退识别描述里的“距离 N 以内”。

规则参考：

- 攻击距离与目标随机判定：`https://games.sina.cn/yxbk/sy/2016-04-27/detail-ifxrpvea1257505.d.html`
- 反击受攻击距离限制：`https://news.17173.com/z/stzb/content/06112020/074430922.shtml`

## 部队加成规则

当前原型没有城建等级系统，按人数做固定近似：2 人同类为 `5%`，3 人同类为 `10%`。

- 同阵营加成：队伍里同阵营武将达到 2 人及以上时，对这些同阵营武将生效，攻击、谋略、防御、速度 +5% / +10%。
- 同兵种加成：队伍里同兵种武将达到 2 人及以上时，对这些同兵种武将生效。
- 步兵同兵种加成：攻击、防御 +5% / +10%。
- 骑兵同兵种加成：攻击、速度 +5% / +10%。
- 弓兵同兵种加成：防御、速度 +5% / +10%。
- 战场卡片会显示已触发的 `魏阵营`、`骑兵种` 等标签，战报开局会记录触发情况。

规则参考：

- 同阵营/同兵种加成：`https://news.17173.com/z/stzb/content/06112020/074430922.shtml`

## 战斗公式说明

当前伤害公式不是原版《率土之滨》1:1。

现在的 `dealDamage()` 是原型模拟，使用：

- 攻击 / 谋略
- 防御
- 兵力比例
- 战法倍率
- 增伤 / 减伤状态
- 小范围随机浮动
- 兵种克制倍率

要更接近原版，需要收集原版战报样本，再校准防御曲线、兵力影响、伤害率和各战法专属特例。

## 准备回合与状态规则

当前按公开攻略和官网战法字段实现第一版规则化战斗流程：

- 战前/准备阶段按 `被动 -> 指挥` 结算，同一阶段按武将速度从高到低执行。
- 正式回合按速度行动，单个武将行动内先判定主动战法，再普通攻击，普攻后判定追击战法。
- 带 `N回合准备` 的主动战法会先进入准备队列，倒计时完成后再发动；准备完成时若处于犹豫或主动封锁，则不会发动。
- 目标选择会先按战法有效距离/攻击距离筛选存活目标，再在可选目标里随机抽取。
- 洞察会免疫怯战、犹豫、混乱、暴走、燃烧、降防等负面状态；同类控制已存在时，后续同类控制不叠加。
- 怯战阻止普通攻击，犹豫阻止主动战法，混乱使本回合无法行动，暴走会让攻击类目标在敌我存活单位中随机选取。
- 援护会替友军承受普通攻击；分兵会在普通攻击后对原目标相邻存活单位造成额外攻击伤害。

规则参考：

- 战法发动顺序、准备回合与同类冲突：`https://www.taptap.cn/moment/192227378620007173`
- 战法类型、控制冲突与准备回合说明：`https://www.gamersky.com/handbooksy/202008/1317011.shtml`
- 攻击距离与可选目标随机判定：`https://games.sina.cn/yxbk/sy/2016-04-27/detail-ifxrpvea1257505.d.html`
- 战法字段和复杂状态文本来源：`https://g0.gph.netease.com/ngsocial/community/stzb/cfg/skill_extra.json?gameid=g10`

## 仍待补齐的率土规则

- 称号/组合加成：需要补充官方称号组合数据后再做，目前 `official-data.js` 还没有这类结构化字段。
- 兵种进阶与特性：例如重骑、轻骑、长弓等细分兵种及特性尚未建模。
- 少数战法的专属优先级、特殊目标规则、同回合刷新/覆盖细节仍需按战报样本继续校准。
- 士气、地形、建筑、赛季规则、兵种转换、政策与宝物暂不实现；宝物系统按当前需求明确跳过。

当前兵种克制是原型规则：

- 骑克步
- 步克弓
- 弓克骑
- 攻击方克制防御方：伤害 +15%
- 攻击方被防御方克制：伤害 -10%
