# 0002 战报入口与全屏战报弹层

## 背景

主界面原先在右侧常驻显示战报详情，并且开战按钮按回合推进。随着战斗回放、战后统计和多封战报管理的需求增加，常驻战报面板会挤占战场空间，也无法承载类似游戏内“个人战报列表 -> 单封战报 -> 详情/统计”的层级。

## 决策

- 主界面不再直接展示战报详情，只保留顶部“战报”入口按钮。
- 点击“开战”后一次性模拟到战斗结束，主战场直接呈现结算后的兵力、胜负和“再战”状态。
- 每次完整战斗生成一封个人战报，未读战报数量显示在“战报”按钮徽标上。
- 点击战报入口打开覆盖全屏的战报弹层，先显示个人战报列表；选择某一封后进入单封战报。
- 单封战报内保留三个底部入口：`战报详情`、`统计`、`阵容详情`。当前实现中：
  - `战报详情` 使用既有逐条战报日志样式。
  - `统计` 拆成 `武将统计` 和 `战法统计` 两个视图。
  - `阵容详情` 先保留为不可用入口，等待后续阵容快照需求。

## 边界

- 战斗引擎仍只负责生成战斗状态、日志和结算结果，不接触 DOM。
- 战报列表、未读状态、全屏弹层和统计表都属于 `src/app.js` 的浏览器状态和渲染职责。
- 战斗状态事件仍进入战报；抽卡、拆解、自动整备等账号/系统事件继续进入系统消息。
- 战报持久化只保存可序列化快照，不保存战斗引擎运行时的循环引用或函数。

## 验证

相关浏览器烟测覆盖：

- 开战后一次性结算并生成 1 封未读战报。
- 打开战报弹层并进入单封战报后，未读徽标清零。
- 战报详情中仍渲染逐条日志。
- 统计视图可以渲染武将统计和战法统计。
