# 0007. SLG V1 采用回合制小地图并复用战斗引擎

## 背景

战斗框架已经能支持三人编队、战法、守军、战报和回放。下一阶段目标是从单场 PVE 原型扩展到单机 SLG：四个势力争夺资源、征兵发展并统一全国。

## 决策

SLG 第一版采用：

- 回合制推进，而不是实时倒计时。
- `25x25` 小型方格世界，而不是先做大州域。
- 四势力固定出生，每个势力一个 `3x3` 主城。
- 资源点随机分布，分为粮草、木材、石料。
- 资源点和主城战斗复用现有战斗引擎。
- SLG 模拟状态、地图 UI 和浏览器接线分模块实现。

## 原因

- 回合制更适合单机原型，能把暂停、存档、AI 和资源结算复杂度压低。
- 小地图能更快验证“扩张、争夺、发展、统一”的策略循环。
- 复用战斗引擎可以让已有编队、战法、战报和 AI 配将继续产生价值。
- 新增独立 `slg-*` 模块能避免把长期世界状态塞进 `battle-engine.js` 或继续膨胀 `app.js`。

## 影响

- 首版不会实现行军时间、要塞、同盟、外交、政策、科技、多队列或赛季规则。
- AI 首版只做本地贪心决策：征兵、升级、攻打相邻高价值目标。
- 如果后续要做半实时或更接近率土的行军表现，应先扩展 SLG 模拟层的命令队列，而不是改战斗引擎。
