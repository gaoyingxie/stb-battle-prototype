# 0009. SLG V2 采用完整军团和命令队列

## 背景

V1 已经能完成“发展、扩张、争夺资源、攻灭主城”的单机 SLG 闭环，但出征仍然是从势力总兵池临时抽出一支三人队伍。这样能快速验证地图循环，却缺少中期策略压力：玩家不用考虑部队位置、连续作战损耗、前线补给或攻守轮换。

下一阶段希望补强部队经营，让玩家围绕多支军团、体力、伤兵、驻守和要塞做决策，同时继续复用已有三人编队战斗引擎。

## 决策

SLG V2 采用完整军团模型：

- 玩家拥有多支部队，主城等级 1-5 分别支持 1/2/3/4/5 支部队。
- 每支部队仍是三人编队，单队上限 `30000`，单武将上限 `10000`。
- 部队拥有位置、体力、当前兵力、伤兵、编队配置和当前命令。
- 占地、攻资源、攻城、返回、驻守和建要塞都进入 `state.slg.commands` 命令队列。
- 要塞作为前线补给点，提供驻扎、返回、体力恢复和伤兵恢复，不提供资源产出。
- AI 使用抽象同规则：AI 也有部队、体力、伤兵和命令队列，但不做完整可视化管理 UI。

## 原因

- 完整军团能把地图层的选择从“点哪里打哪里”推进到“哪支队伍在什么位置、是否还能连续作战、前线是否需要补给”。
- 命令队列比即时结算更适合后续扩展行军时间、驻守拦截、要塞和前线威胁提示。
- 保留三人编队战斗引擎可以继续复用武将、战法、战报和回放，不把长期世界状态塞进 `battle-engine.js`。
- AI 抽象同规则能让敌方行动受到相同资源和恢复约束，同时避免为原型阶段投入完整 AI 军团 UI 成本。

## 影响

- `factions[factionId].armyTroops` 保留为势力预备兵池，不再直接代表每次出征的部队兵力。
- 未来实现需要在 `state.slg` 下新增 `armies` 和 `commands`，并让地块支持 `garrisonArmyIds`。
- 军团生命周期、命令推进、体力恢复、伤兵恢复和 AI 军团决策应落在 `src/slg-world.js` 或后续拆出的 SLG 纯模拟模块。
- `battle-engine.js` 继续只接收三人队伍和兵力快照，不接入 DOM、地图、命令队列或 localStorage。
- UI 应在天下主视图增加“军团”管理区；战斗结果仍进入战报，调动、驻守和建要塞结果进入系统消息。
- 如果需要升级存档版本，可以重置旧 SLG 本地存档；当前项目不为错误旧状态增加复杂兼容分支。

## 暂不包含

- 不做多人实时行军。
- 不做外交、同盟、赛季政策或完整官职系统。
- 不做独立部队动画或地图路径可视化的精细表现，除非后续 UI 迭代专门处理。
- 不改变现有战斗公式、战法语义或武将官方数据导入方式。
