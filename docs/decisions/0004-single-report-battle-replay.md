# 0004 单封战报驱动战况回放

## 背景

原版《率土之滨》的战报可以进入战况回放，用更接近战场表现的方式复看每回合行动。当前原型已经有完整文字战报、武将快照和逐条日志，但此前“战况回放”入口只跳到文字详情。

## 决策

- 第一版战况回放做成战报弹窗内的 2.5D 表现层，不引入 3D 引擎或新外部依赖。
- 每封战报保存自己的 `initialPlayer` / `initialEnemy` 开战快照，回放只播放当前这一封战报的日志。
- 平局续战会产生新的战报；每封继战战报从该封战报自己的承接兵力开始播放，不把多封战报拼成一个长回放。
- 回放时间轴由 `battle.log` 生成，播放状态、进度、倍速、行动高亮、伤害和治疗飘字都属于 `src/battle-replay-ui.js` 的 DOM 表现职责；`src/report-ui.js` 只负责弹窗入口和视图路由。
- 战斗引擎不反向接收回放状态，也不为了动画重跑随机战斗；战报快照才是回放事实源。

## 影响

- 新战报可以稳定复看同一场战斗表现，不会因为重新模拟产生不同结果。
- 旧战报如果缺少开战快照，仍可显示文字详情；新回放能力以新增战报为准。
- 后续如果要升级为 canvas 或 3D 战场，也应继续以单封战报快照和日志作为输入，而不是把动画逻辑放回战斗引擎。
