# 0001. Agent 知识分层

日期：2026-05-13

## 状态

已采纳

## 背景

项目里有几类老化速度不同的知识：

- 给 Agent 和新贡献者看的入口导航。
- 模块边界、验证命令、战斗规则、AI 推荐行为等稳定系统事实。
- 解释某些设计为什么不能被随意“优化”的决策原因。
- 应该可执行、而不是靠猜的本地环境准备步骤。

如果把这些内容都塞进一个大型入口文件，后续会更难导航，也更容易误读。

## 决策

采用四层仓库知识结构：

- `AGENTS.md` 是简洁入口和路由层。
- `docs/` 是稳定系统事实来源。
- `docs/decisions/` 保存带日期的决策记录，用来解释原因和约束。
- `npm run bootstrap` 是稳定的本地进场入口。

## 影响

- 新 Agent 应该先读 `AGENTS.md`，再按链接进入最小相关文档。
- 架构、数据、规则、AI、验证路径等稳定事实应放在 `docs/`，不放在 `AGENTS.md`。
- 非显然约束应该进入决策记录，而不是只留在聊天记录里。
- 环境准备通过 bootstrap 命令自动化，避免 Agent 猜依赖安装或浏览器安装步骤。
