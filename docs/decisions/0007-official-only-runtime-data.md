# 0007 运行态武将和战法只来自官方数据

## 背景

早期原型在 `src/seed-data.js` 中维护过少量种子武将和手写战法，用来支撑页面、抽卡、初始编队和战斗演示。官方资料导入稳定后，这些本地条目会带来两套 id、两套字段来源和额外去重逻辑。

## 决策

`src/seed-data.js` 不再保存任何手写武将或战法，只提供运行态 `HEROES` / `SKILLS` 数组容器。应用启动时由 `official-data.js` 重建这两个数组：

- 官方战法复制到 `SKILLS`，并在运行态接入官方描述推断和必要的专属结算逻辑。
- 官方武将复制到 `HEROES`，缺少自带战法引用的条目跳过。
- 同名、同阵营、同兵种、同自带战法的官方重复武将只保留一个运行态条目。
- 存档规范化、开局阵容、抽卡和 AI 推荐都只使用官方 id。

## 影响

后续不要为了兜底把武将或战法对象写回 `seed-data.js`。缺数据时刷新官方抓取脚本，缺战斗表现时扩展官方战法行为推断、战斗规则或状态系统。旧本地 id 不做兼容迁移；如果本地存档来自旧版本，按现有规范化逻辑重置到官方 id。
