# 0005 战法画像先于实战校准

## 背景

AI 配队已经从单武将面板评分扩展到站位、阵容协同和战法适配，但战法理解仍分散在 `team-ai.js` 的文本推断里。继续调权重前，需要先让 AI、UI 和未来离线基准脚本共享同一套战法画像。

## 决策

先新增 `src/skill-taxonomy.js` 自动派生全量运行态战法画像，并让 `team-ai.js` 优先读取结构化画像。实时自动整备暂不接入实战模拟；实战校准后续作为离线脚本或基准对局使用同一套画像来迭代算法。

## 原因

- 自动整备需要保持即时反馈，不能在用户点击时跑多轮模拟。
- 结构化画像能先解决战法识别一致性问题，避免 AI、UI 和离线工具各自重复解析描述文本。
- `official-data.js` 是生成物，画像只能运行时派生或由独立覆盖表补充，不能手工写回官方数据。

## 后续

如果要继续提升 AI 质量，优先补 `skill-taxonomy.js` 的规则和测试；如果需要实战校准，新增离线 benchmark 脚本读取同一套画像并输出权重调整依据。
