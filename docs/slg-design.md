# 单机 SLG 大地图设计

本文记录《率土之滨》风格单机 SLG 层的规则。目标不是一次性复刻完整赛季，而是在现有三人编队战斗框架之上，先做出“发展、争夺资源、扩张、统一全国”的可玩闭环，再逐步补上军团经营、行军调度和前线补给。

## V1 核心循环

1. 玩家在大地图上查看资源、兵力、主城等级和周边地块。
2. 玩家消耗粮草征兵，消耗木材和石料升级主城。
3. 玩家从相邻己方领地出征，铺开空地、攻打中立资源点或敌方主城中心。
4. 战斗复用现有三人编队战斗引擎；胜利后占领目标，失败则维持原归属。
5. 点击结束回合后，所有存活势力获得资源产出，三个 AI 依次补兵、升级、扩张。
6. 玩家攻陷其他三个势力主城后胜利；玩家主城中心被攻陷后失败。

## 主视图

- 当前 SLG 版本以“天下”大地图作为主视图，页面加载后不再展示旧的单场 PVE 战斗预览面板。
- 玩家出征仍会复用三人编队战斗引擎，但战斗结果只通过战报入口查看，不在主界面常驻双方阵列面板。
- 战报入口放在天下资源栏动作区，和重置天下、结束回合属于同一层级操作。
- 大地图需要优先表达势力归属：已占地块按势力色染色，玩家领地使用更亮的蓝色边界，敌方领地使用各自势力色，未占空地保持低饱和底色。
- 地块详情面板需要直接说明当前地块是否可占领、可出征、可攻城或只是后续铺路跳板，避免玩家只靠试错理解规则。

## 世界和势力

- 地图尺寸固定为 `25x25` 方格。
- 共有四个势力：玩家、北府、西凉、东吴。
- 每个势力初始只有一个主城，主城占 `3x3` 方格，中心格是胜负关键目标。
- 首版允许占领与己方领地正交相邻的空地，用来铺出可完成一局的扩张路径。
- 资源点仍需要与己方领地正交相邻；敌方主城中心只要其 3x3 城域外圈贴住己方领地即可攻城。
- V1 不做外交、同盟、行军时间、铺路冷却、要塞、具体部队队列 UI 或多人实时逻辑。

## 资源和主城

资源类型为粮草、木材、石料。

- 初始资源：粮草 `1200`、木材 `800`、石料 `800`。
- 主城等级：`1-5`。
- 主城每回合产出三种资源，等级越高产量越高。
- 粮草用于征兵，兑换比例为 `1 粮草 = 10 兵力`。
- 木材和石料用于升级主城。
- 资源点分为粮草、木材、石料三种，等级越高产出越高，且守军越强。
- V1 势力总兵力没有 3 万上限，可通过持续征兵累积；`30000` 是单支出征队伍的兵力上限。
- 天下资源栏同时展示当前库存和“每回合产量”，该产量由主城等级基础产出和已占资源点累加而来。
- 地块详情需要展示资源点的精确单回合产量；主城详情需要展示当前主城产量、征兵新增兵力和下级升级成本/产量变化，避免玩家只能靠试错理解经济节奏。

## 战斗和占领

- 玩家出征使用当前编队和战法配置。
- AI 和中立守军复用 `team-ai` 生成三人队伍。
- SLG 战斗通过现有战斗引擎生成战报，进入已有战报列表。
- V1 SLG 兵力是势力长期总兵池；单次出征最多从总兵池抽调 `30000` 组成三人队伍，战后只扣除参战队伍损失，未参战预备兵保留在势力总兵力中。
- 占领空地不会产出资源，也不会生成战报，只作为后续出征的领地跳板。
- 攻下资源点后，该资源点每回合为占领方产出对应资源。
- 攻下敌方主城中心后，该势力覆灭，其主城和已占资源归胜利方。

## V1 AI 回合

每个 AI 每回合最多执行一次主要出征行动，顺序为：

1. 若粮草足够，先征兵累积势力总兵力。
2. 若资源足够且主城未满级，再升级主城。
3. 从相邻可攻击目标中选择价值最高的目标。
4. 资源点价值按等级、资源类型和守军强度评估；主城中心价值更高，但兵力不足时会明显降权。

## V2 军团经营目标

V2 的目标是把“单次从总兵池抽一队出征”升级为“完整军团经营”：玩家需要管理多支部队的位置、体力、伤兵和命令队列，在扩张、补给、驻守和攻城之间做取舍。

- V2 仍保持单机回合制，不引入多人实时逻辑。
- 战斗仍复用现有三人编队战斗引擎，SLG 层只负责把某支部队的武将、战法、兵力和伤兵快照交给战斗引擎。
- `factions[factionId].armyTroops` 保留为势力预备兵池，用于征兵、补员和伤兵恢复；它不再直接代表每次出征可投入的唯一长期战力。
- 玩家拥有完整军团管理 UI；AI 使用同一套体力、伤兵和命令约束，但决策和表现保持抽象。
- 旧 V1 本地世界进度不会迁移；缺少 V2 `armies` / `commands` 字段的 SLG 存档会重置为新的军团状态。

## V2 军团核心循环

1. 玩家在天下界面查看军团列表、部队位置、体力、兵力、伤兵和当前命令。
2. 玩家在主城或要塞整备部队：调整三人编队、补员、恢复伤兵，或让低体力部队休整。
3. 玩家从部队当前位置或驻扎点下达命令：占领空地、攻打资源点、攻城、返回、驻守或建造要塞。
4. 回合推进时先结算资源和恢复，再推进命令队列；命令到达后触发占领、战斗、驻守或要塞建造进度。
5. 胜利部队停留在目标地块，失败部队回到出发地或最近己方主城/要塞；部队保留战后兵力和伤兵。
6. 玩家通过前线要塞和轮换部队持续推进，最终攻陷其他三方主城。

## V2 部队和上限

- 主城等级决定可拥有部队数：1 级 `1` 支、2 级 `2` 支、3 级 `3` 支、4 级 `4` 支、5 级 `5` 支。
- 每支部队仍是三人编队，使用现有前锋、中军、大营站位和两格可配战法。
- 单支部队总兵力上限保持 `30000`；单名武将兵力上限保持 `10000`。
- 部队可缺兵出征，但实际战力按当前兵力进入战斗；空位或 0 兵武将不参与战斗。
- 玩家部队需要唯一武将身份，同名同身份武将不能同时出现在多支部队中；后续如果要支持重复卡，应先明确“同名不同卡”的身份规则。
- AI 部队也受数量、体力和伤兵约束，但队伍生成可继续复用 `team-ai`，不要求暴露完整配将 UI。

## V2 体力和休整

- 每支部队体力上限为 `100`。
- 占领空地、调动、返回消耗 `20` 体力。
- 攻打资源点或敌方主城中心消耗 `25` 体力。
- 在主城或要塞驻扎时，每回合恢复 `20` 体力。
- 在野外驻守时，每回合恢复 `10` 体力。
- 体力不足时不能下达对应命令；当前命令已经在执行的部队不会因为中途体力不足而取消。

## V2 伤兵、补员和预备兵

- 战斗结束后，部队保留剩余兵力和伤兵，不再只把结果折算回势力总兵池。
- 主城和要塞可恢复伤兵；恢复需要消耗粮草，并从势力预备兵池中补足可恢复部分。
- 默认恢复比例为每回合恢复该部队伤兵的 `35%`，恢复上限不超过武将 `maxTroops`。
- 如果预备兵或粮草不足，优先按前锋、中军、大营顺序补齐当前部队的可用兵力，再保留未恢复伤兵。
- 部队在野外驻守不会自动恢复伤兵，只恢复少量体力。
- 阵亡到 0 兵的武将仍保留在部队配置中，但需要回到主城或要塞补员后才能再次参战。

## V2 命令队列和行军

- `state.slg.commands` 记录所有进行中的命令。每条命令至少包含 `id`、`type`、`factionId`、`armyId`、`originTileId`、`targetTileId`、`remainingTurns` 和 `status`。
- 命令类型包括 `march`、`attack`、`return`、`garrison`、`buildFort`。
- V2 仍然是回合制行军：相邻地块命令默认 `1` 回合到达；跨多格调动按曼哈顿距离折算，最低 `1` 回合。
- 部队有进行中命令时不能再次下令；命令完成后更新部队位置、体力、兵力、伤兵和地块归属。
- 占领空地、攻打资源点和攻城都通过命令队列结算，不再点击后立即完成。
- 地块可记录 `garrisonArmyIds`，用于表示正在驻守的部队；驻守部队会参与该地块被攻击时的防守兵力计算。

## V2 要塞和前线补给

- 要塞建在己方已占空地上，不能建在主城、资源点或敌方领地上。
- 建造要塞消耗木材和石料，默认成本为木材 `600`、石料 `500`。
- 建造要塞耗时 `2` 回合，建造期间占用执行命令的部队。
- 要塞可作为部队驻扎点、返回点、体力恢复点和伤兵恢复点。
- 要塞本身不产出资源，也不改变胜负条件；它的价值是缩短前线补给距离。
- 如果要塞所在地块被敌方占领，该要塞失效，驻扎部队按失败撤退规则回到最近己方主城或要塞。

## V2 AI 规则

- AI 使用抽象同规则：它拥有部队、体力、伤兵、预备兵和命令队列，但 UI 只展示势力概览、前线威胁和可见驻守。
- AI 每回合先恢复驻扎部队，再按资源状态决定征兵、升级或建造前线要塞。
- AI 选择行动时综合评估资源点等级、前线距离、目标守军、己方体力、伤兵比例和敌方主城威胁。
- AI 优先避免让低体力或重伤部队连续攻坚；当主城附近受到威胁时，优先调回部队驻守。
- AI 可继续用 `team-ai` 生成或刷新部队配置，但同一回合只推进有限数量的命令，避免玩家被不可读的 AI 连续行动淹没。

## V2 UI 入口

- 天下主界面新增“军团”管理区，和资源栏、势力概览、地块详情同属 SLG 主视图。
- 军团列表展示：部队名、当前位置、当前命令、体力、总兵力、伤兵、三名武将和可用操作。
- 地块详情根据选中部队和目标地块显示可执行命令：出征、调动、返回、驻守、建要塞或休整。
- 地图地块需要显示正在行军、驻守和建造中的状态，避免玩家不知道命令为何还未结算。
- 战报入口继续承接所有攻打资源点和攻城战斗；占空地、调动、驻守和建要塞进入系统消息。
- 现有编队面板编辑当前选中的玩家军团；切换军团前保存当前三人编队，切换后载入目标军团的编队。

## V2 已实现状态模型

- `state.slg.armies` 管理所有部队。部队记录 `id`、`factionId`、`name`、`homeTileId`、`locationTileId`、`status`、`stamina`、`troops`、`wounded`、`formation` 和 `currentCommandId`。
- `state.slg.commands` 管理命令队列。命令记录 `type`、起点、目标、剩余回合、状态和结算结果。
- `tile.garrisonArmyIds` 管理驻守关系；没有驻守时为空数组。
- `tile.fort` 管理要塞关系；要塞只存在于己方已占空地，提供驻扎、返回、体力恢复和伤兵恢复，不提供资源产出。
- `factions[factionId].armyTroops` 继续表示预备兵池，不直接进入战斗；战斗只读取具体 `army` 的当前兵力和伤兵。
- 如果落地时需要升级存档版本，优先使用干净的新状态结构；这个原型允许重置旧 SLG 本地存档，不为错误旧状态堆复杂迁移分支。

## 模块边界

- `src/slg-rules.js`：SLG 常量、资源、势力、产出、升级和守军数值。
- `src/slg-world.js`：纯状态模拟，负责地图生成、产出、征兵、升级、AI 回合、占领、军团命令和胜负判断。
- `src/world-ui.js`：大地图 DOM 渲染、战报入口和按钮事件派发，不直接修改状态。
- `src/army-ui.js`：军团管理区 DOM 渲染、选中军团和补员按钮派发，不直接修改状态。
- `src/app.js`：把 SLG 状态接入 localStorage、战斗引擎、战报、系统消息和浏览器事件。
- `styles/world.css`：大地图、资源栏、地块、势力概览和地块详情样式。
- `styles/army.css`：军团管理区、军团卡、体力/兵力/伤兵摘要和补员入口样式。
- 军团生命周期仍属于 `src/slg-world.js`，不要塞进 `battle-engine.js`。

## 验证路径

- `npm run check`：语法检查。
- `npm run test:slg`：纯模拟规则回归。
- `npm run smoke:browser`：浏览器加载、大地图主视图渲染、征兵、占领、回合推进和战报入口烟测。
- V2 军团覆盖：主城等级对应部队上限、体力消耗和恢复、伤兵恢复、命令到达、要塞补给、驻守防守、AI 使用抽象部队出征、军团 UI 渲染和命令按钮流。
