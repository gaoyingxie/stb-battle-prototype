# 单机 SLG 大地图 V1 设计

本文记录《率土之滨》风格单机 SLG 层的第一版规则。目标不是一次性复刻完整赛季，而是在现有三人编队战斗框架之上，先做出“发展、争夺资源、扩张、统一全国”的可玩闭环。

## 核心循环

1. 玩家在大地图上查看资源、兵力、主城等级和周边地块。
2. 玩家消耗粮草征兵，消耗木材和石料升级主城。
3. 玩家从相邻己方领地出征，铺开空地、攻打中立资源点或敌方主城中心。
4. 战斗复用现有三人编队战斗引擎；胜利后占领目标，失败则维持原归属。
5. 点击结束回合后，所有存活势力获得资源产出，三个 AI 依次补兵、升级、扩张。
6. 玩家攻陷其他三个势力主城后胜利；玩家主城中心被攻陷后失败。

## 世界和势力

- 地图尺寸固定为 `25x25` 方格。
- 共有四个势力：玩家、北府、西凉、东吴。
- 每个势力初始只有一个主城，主城占 `3x3` 方格，中心格是胜负关键目标。
- 首版允许占领与己方领地正交相邻的空地，用来铺出可完成一局的扩张路径。
- 资源点仍需要与己方领地正交相邻；敌方主城中心只要其 3x3 城域外圈贴住己方领地即可攻城。
- 首版不做外交、同盟、行军时间、铺路冷却、要塞、多队列或多人实时逻辑。

## 资源和主城

资源类型为粮草、木材、石料。

- 初始资源：粮草 `1200`、木材 `800`、石料 `800`。
- 主城等级：`1-5`。
- 主城每回合产出三种资源，等级越高产量越高。
- 粮草用于征兵，兑换比例为 `1 粮草 = 10 兵力`。
- 木材和石料用于升级主城。
- 资源点分为粮草、木材、石料三种，等级越高产出越高，且守军越强。

## 战斗和占领

- 玩家出征使用当前编队和战法配置。
- AI 和中立守军复用 `team-ai` 生成三人队伍。
- SLG 战斗通过现有战斗引擎生成战报，进入已有战报列表。
- SLG 兵力是长期状态；出征时会按总兵力分配到三人编队，战后剩余兵力写回势力。
- 占领空地不会产出资源，也不会生成战报，只作为后续出征的领地跳板。
- 攻下资源点后，该资源点每回合为占领方产出对应资源。
- 攻下敌方主城中心后，该势力覆灭，其主城和已占资源归胜利方。

## AI 回合

每个 AI 每回合最多执行一次主要出征行动，顺序为：

1. 若粮草足够且兵力未满，先征兵。
2. 若资源足够且主城未满级，再升级主城。
3. 从相邻可攻击目标中选择价值最高的目标。
4. 资源点价值按等级、资源类型和守军强度评估；主城中心价值更高，但兵力不足时会明显降权。

## 模块边界

- `src/slg-rules.js`：SLG 常量、资源、势力、产出、升级和守军数值。
- `src/slg-world.js`：纯状态模拟，负责地图生成、产出、征兵、升级、AI 回合、占领和胜负判断。
- `src/world-ui.js`：大地图 DOM 渲染和按钮事件派发，不直接修改状态。
- `src/app.js`：把 SLG 状态接入 localStorage、战斗引擎、战报和系统消息。
- `styles/world.css`：大地图、资源栏、地块、势力概览和地块详情样式。

## 验证路径

- `npm run check`：语法检查。
- `npm run test:slg`：纯模拟规则回归。
- `npm run smoke:browser`：浏览器加载、大地图渲染、征兵、占领、回合推进和战报入口烟测。
