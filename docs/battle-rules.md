# 战斗规则说明

当前模拟器不是《率土之滨》服务端 1:1 复刻，而是面向原型验证的近似模型。伤害曲线、伤兵池和部分战法效果已经规则化，但仍需要真实战报样本继续校准。

## 战斗流程

1. 创建双方单位：大营、中军、前锋各 10000 兵。
2. 应用同阵营和同兵种编队加成。
3. 准备回合结算：先被动，再指挥；同阶段按速度从高到低执行。
4. 正式回合开始：结算灼烧等回合开始效果。
5. 每回合按速度行动；速度包含基础速度、先手加成和小随机值。
6. 单个武将行动内：先处理准备完成的主动战法，再判定主动战法，再普通攻击，普通攻击后判定追击战法。
7. 回合结束时状态持续回合数减少。
8. 一方大营兵力归零时战斗结束；满 8 回合后按总兵力差判定胜负，差值小于 1200 视为平局。

## 状态效果

| 状态 | 类型 | 当前效果 |
| --- | --- | --- |
| 攻击提高 | 属性 | 提高攻击属性，参与攻击伤害计算。 |
| 谋略提高 | 属性 | 提高谋略属性，参与策略伤害和治疗量计算。 |
| 防御提高 | 属性 | 提高防御属性，降低受到的攻击伤害。 |
| 防御降低 | 负面属性 | 降低防御属性，使受到的攻击伤害提高。 |
| 先手 | 行动 | 提高行动速度，影响行动顺序。 |
| 攻击距离提高 | 距离 | 增加普通攻击距离。 |
| 攻击距离降低 | 负面距离 | 降低普通攻击距离。 |
| 造成伤害提高 | 伤害 | 攻方增伤乘区。 |
| 受到伤害降低 | 伤害 | 守方减伤乘区，最低保留 15% 伤害。 |
| 受到伤害提高 | 负面伤害 | 守方易伤乘区。 |
| 规避 | 防御 | 受到伤害前按概率完全规避本次伤害。 |
| 灼烧 | 负面伤害 | 回合开始造成固定兵损，并进入伤兵池。 |
| 怯战 | 控制 | 无法普通攻击。 |
| 犹豫 | 控制 | 无法发动主动战法；准备完成的主动战法也会失效。 |
| 混乱 | 控制 | 本回合无法行动。 |
| 暴走 | 控制 | 攻击类目标在敌我存活单位中随机选择。 |
| 洞察 | 免疫 | 免疫负面状态。 |
| 援护 | 防御 | 替友军承受普通攻击。 |
| 分兵 | 攻击 | 普通攻击后，对原目标相邻存活单位造成额外攻击伤害。 |
| 反击 | 攻击 | 受到普通攻击后按概率反击，仍需满足攻击距离。 |
| 主动封锁 | 负面光环 | 行动时按概率使主动战法发动失败。 |
| 怯战光环 | 负面光环 | 行动时按概率使普通攻击失败。 |

洞察会免疫负面状态。怯战、犹豫、混乱、暴走属于同类控制，同一控制已存在时，后续同类控制不会叠加。

## 攻击距离

战场按 6 个压缩位置计算距离：

```text
我方大营 0 -> 我方中军 1 -> 我方前锋 2 -> 敌方前锋 3 -> 敌方中军 4 -> 敌方大营 5
```

普通攻击只能选择距离小于等于当前攻击距离的存活敌军。若战斗中前排阵亡，会按双方仍存活单位重新压缩前后关系，避免空位永久拉长距离。

攻击距离来自武将自身的 `distance` 字段，不由兵种直接推导。`official-data.js` 里的官方武将记录都带有武将级攻击距离；只有极少数早期手写种子数据缺失该字段时，原型才临时按 `2` 兜底，避免界面和模拟器无法运行。这个兜底值不是《率土之滨》规则。

反击也检查攻击距离；距离不足时只记录触发失败。

| 克制关系 | 伤害修正 |
| --- | --- |
| 骑克步 | +15% |
| 步克弓 | +15% |
| 弓克骑 | +15% |

## 编队加成

当前原型没有城建、称号、兵种进阶等系统，因此同阵营和同兵种只做固定近似：

- 2 名同阵营：同阵营成员攻击、谋略、防御、速度 +5%。
- 3 名同阵营：同阵营成员攻击、谋略、防御、速度 +10%。
- 2 名同兵种：触发该兵种加成 +5%。
- 3 名同兵种：触发该兵种加成 +10%。

兵种加成属性：

| 兵种 | 加成属性 |
| --- | --- |
| 步 | 攻击、防御 |
| 骑 | 攻击、速度 |
| 弓 | 防御、速度 |

## 伤害公式

公式实现位于 `src/battle-rules.js` 的 `calculateDamageFormula()`，战报会记录关键中间项。

输入项：

- `rate`：战法伤害率，例如普通攻击为 0.62。
- `offense`：攻击伤害取攻击属性，策略伤害取谋略属性。
- `attackerTroops`：攻方当前兵力。
- `defenderDefense`：守方防御，包含防御提高/降低。
- `defenderStrategy`：守方谋略，用于抵消策略伤害。
- `armMultiplier`：兵种克制乘区。
- `damageUp`：攻方增伤。
- `damageTakenUp`：守方易伤。
- `damageDown`：守方减伤。
- `variance`：0.96 到 1.04 的随机浮动。

核心计算：

```text
兵力曲线 = 300 * 攻方当前兵力 / (3500 + 攻方当前兵力)

攻击基础压强 = 373 * 攻方当前兵力 / (7700 + 攻方当前兵力)
策略基础压强 = 178 * 攻方当前兵力 / (6459 + 攻方当前兵力)

策略抵消 = min(42%, max(0, (守方谋略 - 80) / 520))
有效攻方属性 = max(20, 攻方属性 * (1 - 策略抵消))

攻击防御修正 = 100 / (100 + max(0, 守方防御 - 65) * 0.62)
策略防御修正 = 1

初始伤害 = 基础压强 + 伤害率 * 兵力曲线 * 有效攻方属性 * 0.075
最终伤害 = 初始伤害
  * 防御修正
  * (1 + 攻方增伤)
  * (1 + 守方易伤)
  * max(15%, 1 - 守方减伤)
  * 兵种克制
  * 随机浮动

最终兵损 = max(60, round(最终伤害))
```

这套公式是当前原型曲线，不代表原版服务端公式。要继续靠近原版，需要用真实战报样本校准防御曲线、兵力影响、战法专属倍率和特殊优先级。

## 伤兵与治疗

- 受到兵损时先扣当前可战兵力。
- 实际兵损的 95% 进入伤兵池。
- 武将兵力归零时，伤兵池保留 60%。
- 治疗只能从伤兵池恢复，不能恢复已经彻底损失的兵力。
- 治疗量上限同时受当前伤兵池和最大兵力缺口限制。

## 战法发动

- 指挥/被动：准备回合触发。
- 主动：行动阶段按发动率判定。
- 追击：普通攻击后按发动率判定。
- 带准备回合的主动战法会进入准备队列；倒计时完成后再发动。
- 准备完成时如果处于犹豫或主动封锁，本次战法不会发动。

官方 `probability` 如果是 `25%-35%` 这类区间，战斗判定按右侧满级值处理；弹窗仍保留原始区间，并显示“满级按 X%”。

- 称号/组合加成。
- 细分兵种、兵种进阶和兵种特性。
- 士气、地形、城建、赛季规则、政策、宝物。
- 少数战法的专属优先级、覆盖规则、目标规则和特殊例外。
- 与原版战报完全一致的服务端伤害曲线。
